## Computational Thinking -JAVA PROJECTS (Traffic Flow, Stock Market, or any Simulator Project)
 - Total: 100 points
 - Option 1: Design your own projects as shown in the examples pdf file.
 - Option 2: Use the examples as the skeleton project but add your own thoughts to it.
 - Final project presentation and defense.
   a. You will present your project on the final day.
   b. You will use the GUI template for your code. You may modify the template class methods if you have different method.
   c. If you use your own project rather than from the examples, you will create a different GUI class using the provided one as an example.
   

## Category,Points,CED Alignment,Criteria (Projects must meet the following requirements)
1. Design Document & Pseudocode|,15|,Practice 1 (Design)|,"Clear problem statement, student-created UML(draw or use draw.io) , complete pseudocode for simulation rules, parameter experiments explained."

3. OOP Design & Class Creation|,15|,Unit 3,"Two custom classes with proper encapsulation, constructors, and meaningful methods."
4. Data Structures & Algorithms|,15|,Unit 4,ArrayList used effectively; simulation loop + data analysis algorithms implemented.
5. Control Structures & Logic|,8|,Unit 2,"Appropriate if/else, loops, and Boolean logic for rules and decisions."
6. "Objects, Methods & Foundations"|,5|,Unit 1,"Proper use of objects, Math class, Scanner/PrintWriter."
7. Documentation & Development Log|,10|,Practice 4,"Javadoc-style comments, full development log with AI disclosures (if used), README with UML."
8. "Demo, Analysis & Reflection"|,7|,All Practices,Live demo + explanation of design decisions; reflection shows computational thinking and real-world connection.

## Grading Notes

1. AI use allowed only if fully documented in development log. 
2. Unmodified AI code without explanation = 0 for entire project.
3. Highest scores go to students who clearly explain why they chose specific simulation rules and what their experiments revealed.
