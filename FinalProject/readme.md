##Computational Thinking Focus (Traffic Flow, Stock Market, or any Simulator Project)
 - Total: 100 points
 - Aligned with 2025-26 CED (Units 1–4 + Practices)

## Category,Points,CED Alignment,Criteria (Student must demonstrate)
1. Design Document & Pseudocode|,15|,Practice 1 (Design)|,"Clear problem statement, student-created UML, complete pseudocode for simulation rules, parameter experiments explained."
2. Code Correctness & Functionality|,25|,"Units 1–4, Practice 2 (Develop)",Program runs without errors; simulation works with user-chosen parameters; edge cases handled.
3. OOP Design & Class Creation|,15|,Unit 3,"Two custom classes with proper encapsulation, constructors, and meaningful methods."
4. Data Structures & Algorithms|,15|,Unit 4,ArrayList used effectively; simulation loop + data analysis algorithms implemented.
5. Control Structures & Logic|,8|,Unit 2,"Appropriate if/else, loops, and Boolean logic for rules and decisions."
6. "Objects, Methods & Foundations"|,5|,Unit 1,"Proper use of objects, Math class, Scanner/PrintWriter."
7. Documentation & Development Log|,10|,Practice 4,"Javadoc-style comments, full development log with AI disclosures (if used), README with UML."
8. "Demo, Analysis & Reflection"|,7|,All Practices,Live demo + explanation of design decisions; reflection shows computational thinking and real-world connection.

Grading Notes

AI use allowed only if fully documented in development log.
Unmodified AI code without explanation = 0 for entire project.
Highest scores go to students who clearly explain why they chose specific simulation rules and what their experiments revealed.
